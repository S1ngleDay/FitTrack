# 🎮 Demo Mode для FitTrack - SUMMARY

## Что было сделано?

Создана полная система демо-режима для загрузки реалистичных тестовых данных в приложение.

---

## 📂 Созданные файлы

### **1. `src/utils/demoData.js`** — Хранилище демо-данных
- 20 полных тренировок за 3 месяца (разные типы)
- 3 готовых плана тренировок
- Полный профиль пользователя
- GPS маршруты для кардио-тренировок
- Силовые упражнения с полной информацией

### **2. `src/hooks/useDemoDataInit.js`** — Инициализация демо-данных
- Хук, загружающий демо-данные при старте приложения
- Проверяет флаг `DEMO_MODE`
- Заполняет Zustand store (workoutStore, userStore)

### **3. Модифицированные файлы**
- **`src/store/workoutStore.js`** — добавлены методы:
  - `addMultipleWorkouts()` — загрузить несколько тренировок
  - `setWorkoutPlans()` — загрузить планы
  
- **`App.js`** — добавлен импорт и вызов `useDemoDataInit()` хука

### **4. Документация**
- `DEMO_MODE.md` — Подробное руководство
- `DEMO_MODE_QUICK_START.md` — Быстрый старт (30 секунд)

---

## 🎯 Как использовать?

### **Включить демо-режим (Вариант 1 - Быстро)**

```javascript
// src/utils/demoData.js
export const DEMO_MODE = true;  // ✅ Включить
// Перезагрузить приложение → готово!
```

### **Выключить демо-режим (Перед релизом)**

```javascript
// src/utils/demoData.js
export const DEMO_MODE = false;  // ❌ Выключить
```

## 🔄 Как это работает?

```
1. Приложение стартует (App.js)
        ↓
2. useDemoDataInit() проверяет DEMO_MODE
        ↓
3. Если DEMO_MODE = true:
   - Загружаем 20 тренировок в workoutStore
   - Загружаем 3 плана в workoutStore
   - Заполняем профиль в userStore
        ↓
4. Данные сохраняются в AsyncStorage
        ↓
5. Приложение показывает данные на экранах
```

## ⚙️ Технические детали

### **Используемые методы:**

| Компонент | Метод | Назначение |
|-----------|-------|-----------|
| `workoutStore` | `addMultipleWorkouts()` | Загрузить 20 тренировок |
| `workoutStore` | `setWorkoutPlans()` | Загрузить 3 плана |
| `userStore` | `updateUser()` | Заполнить профиль |

- AsyncStorage (локальное хранилище React Native)
- Zustand store (управление состоянием)
- Автоматическое сохранение через `persist` middleware

## ❌ Если что-то не работает

### **Проблема: Данные не загружаются**
- Проверьте: `export const DEMO_MODE = true`
- Полный перезагруз (не горячий refresh)
- Очистите кеш приложения

### **Проблема: Консоль пуста**
- Очистите консоль iOS Simulator / Android Emulator
- Перезагрузите приложение
- Смотрите в Expo CLI логи

### **Проблема: Нужны свои демо-данные**
- Отредактируйте `demoDemoWorkouts[]` в `src/utils/demoData.js`
- Измените даты, метрики, упражнения
- Перезагрузите приложение
